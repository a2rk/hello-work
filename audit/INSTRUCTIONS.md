# Как пользоваться этим набором — Legends Module cycle

> TL;DR: один файл-промпт скармливаешь Claude каждую итерацию, он сам выберет следующий таск из «Истории легенд» и сделает один шаг.

---

## Что лежит в `audit/`

| Файл | Что внутри | Кто его трогает |
|---|---|---|
| `CONTEXT.md` | Цель цикла, структура JSON легенд, архитектура, правила, **task ledger** (76 тасков) со статусами `[ ] / [~] / [x]` | Claude правит статусы тасков; ты — если хочешь добавить таск или сменить приоритет |
| `DRIVER.md` | **Промпт**, который ты копируешь в Claude каждую итерацию | Никто не трогает после создания |
| `INSTRUCTIONS.md` | Этот файл — для тебя | Никто не трогает |

---

## Цель этого цикла

Внедрить **модуль «Истории легенд»** — 60 расписаний великих людей (от Франклина до Виллинка), которые юзер может изучать и применять к своим managed apps. Список / Detail / Favorites / Apply / Revert. Все 60 JSON встраиваются в bundle.

Старт: версия `0.10.1`. Финал: `0.11.0` (минорный bump после 64-76 тасков). Каждый таск = одна патч-версия + GitHub Release.

## Workflow

### Каждая итерация

1. Открой Claude Code в этой репе.
2. Скопируй ВЕСЬ текст из `audit/DRIVER.md` (от строки «Ты — senior macOS engineer» до конца) и вставь в чат.
3. Жди отчёт. Claude скажет: ID таска, что сделал, версия релиза, какой следующий.
4. Проверь визуально (`git diff`, `git log`, посмотри файлы которые он правил).
5. Если всё ОК — следующая итерация. Если поплыло — `git restore .` + перезапуск промпта.

### Через `/loop`

Если хочешь батчем — запускай `/loop делай дальше, не ограничивайся одной таской`. Claude в dynamic-mode сам себе ставит `ScheduleWakeup` каждые 2 минуты и работает блоками по 5-10 тасков пока ты не остановишь.

### Нюансы

- **Claude НЕ накапливает контекст между сессиями** — он каждый раз заново читает `CONTEXT.md`. Поэтому ledger со статусами ОБЯЗАТЕЛЬНО должен быть актуальным после каждой итерации; Claude обновит его сам.
- **Каждый второй таск — verify** (по дизайну). Не пропускай — это страховка от регрессий.
- **Релизы — каждый таск** (не пакетами, как было в первом цикле). Это даёт granular rollback и видимый прогресс.
- **Если Claude нашёл что-то новое во время verify** — он создаст follow-up в секции 7. Ledger живой.

## Если что-то идёт не так

### «Claude взял не тот таск»

«Ты должен был взять TASK-LNN, не TASK-LMM. Откати правки через git restore и попробуй ещё раз».

### «Регрессия в собранном бинаре»

Включи dev mode (10 тапов по «Данные»), посмотри Diagnostics. Скинь содержимое в чат.

### «Хочу добавить свой таск»

Допиши руками в `audit/CONTEXT.md` в нужную фазу или в секцию `## 7. Follow-up tasks`. Формат как у остальных. Claude дойдёт по очереди.

### «Хочу пропустить таск»

Помечай `[x] **TASK-LNN [...]** ~skipped: <причина>~`. Claude увидит как done и не возьмёт.

### «Claude закрыл цикл, нужен ещё»

Когда все таски `[x]`, попроси: «Запусти полный audit модуля Legends, ищи слабые места, сгенери Phase J с новыми тасками». Это создаст продолжение.

## Что НЕ делать

- ❌ Не запускай несколько Claude-сессий параллельно над этим ledger'ом — затрут друг друга
- ❌ Не редактируй файлы продакта (`Sources/...`) пока итерация в работе
- ❌ Не делай `git reset --hard` если итерация в `[~]` — потеряешь прогресс
- ❌ Не правь `audit/DRIVER.md` без необходимости — изменит поведение всех будущих итераций

## Ожидаемая динамика

- **Тасков всего**: 76 (38 impl + 38 verify), плюс возможные follow-up'ы.
- **Фаз**: 9 (A — data foundation; B — persistence; C — sidebar; D — list view; E — detail; F — favorites/search; G — apply flow; H — polish; I — final regression).
- **Темп**: 1 итерация = 1 таск. Реалистично 5-15 итераций в день, в режиме `/loop` — батчи по 5-10 за заход.
- **Срок**: 5-15 дней при регулярном прогоне.
- **Версии**: 0.10.1 → 0.10.65 → 0.11.0 (минорный финал).

## Финальная проверка перед стартом

1. У тебя в репо чисто (`git status` зелёный)?
2. Текущая ветка `main`?
3. Self-signing cert «HelloWork Self-Signed» в keychain (`security find-identity -p codesigning ~/Library/Keychains/login.keychain-db | grep "HelloWork Self-Signed"`)?
4. Можешь собрать вручную (`swift build`)?
5. Папка `legens_module/` в корне репо существует с 60 файлами (TASK-L05 будет переносить)?

Если все пять — да, можно стартовать. Поехали.
