# Hello work — мультиязычность (i18n)

## Принцип

Все тексты — в типизированном словаре `Translation`. **Никаких хардкод-строк в вьюхах.** SwiftUI получает текущий словарь через `EnvironmentValue`. AppDelegate (NSMenu) — через `state.t` напрямую.

## Файлы

```
Domain/
├── AppLanguage.swift    enum {system, en, ru, zh} + L10n резолвер
├── Translation.swift    struct с 70+ полями (вся типизация)
└── Translations.swift   три статических инстанса (.en, .ru, .zh)

Theme/
└── EnvironmentTranslation.swift   SwiftUI EnvironmentKey \.t
```

## Как добавить новую строку

### 1. Добавь поле в `Translation`

```swift
struct Translation {
    // ...
    let myNewLabel: String
}
```

Если строка с подстановкой — closure:
```swift
let helloName: (String) -> String  // "Привет, \(name)!"
```

### 2. Заполни в **трёх** инстансах в `Translations.swift`

```swift
static let en = Translation(/* ... */, myNewLabel: "My label", helloName: { "Hello, \($0)!" })
static let ru = Translation(/* ... */, myNewLabel: "Моя метка", helloName: { "Привет, \($0)!" })
static let zh = Translation(/* ... */, myNewLabel: "我的标签", helloName: { "你好,\($0)!" })
```

Компилятор не даст забыть — `Translation` имеет **синтезированный init со всеми параметрами**, любой пропуск ломает билд.

### 3. Используй в SwiftUI-вьюхе

```swift
struct MyView: View {
    @Environment(\.t) var t

    var body: some View {
        Text(t.myNewLabel)
        Text(t.helloName("Игорь"))
    }
}
```

### 4. В AppDelegate / NSMenu — через `state.t`

```swift
let item = NSMenuItem(title: state.t.menuQuit, action: ..., keyEquivalent: "")
```

Менюбар обновляется по таймеру в `refreshMenuTitles()` — смена языка подхватывается за ≤ 0.25с без рестарта.

## Как добавить новый язык

1. Добавь `case` в `AppLanguage`:
   ```swift
   case system, en, ru, zh, ja
   ```

2. Расширь `displayName`:
   ```swift
   case .ja: return "日本語"
   ```

3. Добавь `static let ja = Translation(...)` в `Translations.swift` со всеми полями.

4. Добавь case в `L10n.resolved(_:)`:
   ```swift
   case .ja: return .ja
   ```

5. И в `L10n.resolveSystem()`:
   ```swift
   case "ja": return .ja
   ```

## Как пользователь меняет язык

`SettingsView` → блок «Язык» → `Picker` привязан к `state.language`.

`AppLanguage` сохраняется в `UserDefaults` под ключом `helloWorkLanguage`. Через `didSet` пишется автоматически. На запуске читается обратно в `init()`.

`.system` — берёт `Locale.preferredLanguages.first` и матчит по двухбуквенному коду (`ru`, `zh`, `en` ⇒ fallback). Любой другой язык ОС → `en` (английский — дефолт).

## Что **не** переводится

- Бренд **«Hello work»** — везде латиница, как у Apple / Microsoft и т.д.
- Названия других языков на их же языке (English / Русский / 中文) — чтобы пользователь нашёл свой.
- Технические идентификаторы (bundle ID, имена файлов, `Locale` коды).
- Цифры и единицы дат (через `RelativeDateTimeFormatter` с правильной locale).

## Чек-лист перед коммитом новой фичи с текстом

- [ ] Все строки в `Translation` (нет литералов в вьюхах)
- [ ] Заполнено во всех трёх инстансах (`.en`, `.ru`, `.zh`)
- [ ] Compiler ошибок нет (он не даст забыть поле)
- [ ] В вьюхе используется `@Environment(\.t)` или `state.t`
- [ ] Если строка появляется в menubar → проверь что `refreshMenuTitles()` её обновляет
